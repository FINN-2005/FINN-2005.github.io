The Game Loop

At the heart of every game engine is the game loop. It is the system responsible for updating the world and rendering frames continuously.

Every frame, the engine updates the game state and then draws the result to the screen.

Interactive Visualization

This simulation demonstrates the continuous update and draw cycle used by most games.

Typical Game Loop Structure

init()

setup()

while (gameRunning) {
    update()
    draw()
}

Each stage has a specific responsibility in the game engine.

Stages of the Loop

  • Init – Load resources and initialize systems.
  • Setup – Create objects and initial game state.
  • Update – Process physics, input, and game logic.
  • Draw – Render the current frame to the screen.

Why the Game Loop Matters

The game loop ensures that the simulation runs continuously and consistently. Physics systems, AI, animations, and rendering all rely on this repeating update cycle.

Frameworks like p5.js implement this automatically through the setup() and draw() functions.
